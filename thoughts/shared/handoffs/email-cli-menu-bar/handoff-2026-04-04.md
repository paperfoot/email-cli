## Key Decisions Made

1. **tray-item crate is a dead end for dynamic UX** — `set_icon()` doesn't update after `display()`, no `set_label()`, no separators on macOS. Was replaced with raw `objc2` + `objc2-app-kit`.

2. **Notifications require a codesigned .app bundle on macOS 26** — Standalone binaries silently fail `UNUserNotificationCenter.requestAuthorization()`. TCC requires code signing (ad-hoc is sufficient). The bundle is at `assets/EmailCLI.app/`.

3. **NSUserNotification (old API) shows notifications but clicking opens Script Editor**. UNUserNotificationCenter (new API) requires the .app bundle. We went with UNUserNotificationCenter inside the bundle.

4. **`inbox read` now marks as read by default** (was opt-in). `--no-mark-read` to skip.

5. **Codex GPT-5.4 recommended `objc2` over old `cocoa`/`objc` crates** — cleaner bindings, better maintained. This was the right call.

### CRITICAL BUG: Menu item clicks don't work
The objc2 rewrite compiles and shows the icon + menu correctly, but clicking "Sync Now", "Mark All Read", or "Quit" **does nothing**. The root cause:

**Menu items have no action targets.** They're created with `action: None` (line ~236 in daemon.rs). The `poll_click` function tries to detect clicks by watching `isHighlighted()` state transitions via the `tag` field — this is a hack that doesn't work because:
1. `isHighlighted()` may not behave as expected in NSMenu
2. The NSRunLoop polling at 200ms may miss the highlight → unhighlight transition
3. Menu items without actions are visually disabled (grayed out) in some macOS versions

**The correct approach:** Create Objective-C callback objects (using `objc2::declare_class!`) that handle `@selector(action:)` for each menu item. Or use `NSMenuItem::setTarget()` + `NSMenuItem::setAction()` with a custom ObjC class.

## Gotchas & Warnings

- **Don't use `tray-item` crate** — We already proved it can't do dynamic updates. Don't go back to it.
- **macOS 26 requires ad-hoc codesigning** for .app bundles to show notifications. Run `codesign --force --deep --sign - assets/EmailCLI.app` after any modification to bundle contents.
- **`NSRunLoopMode::Default`** may not be the right constant — the current code uses `NSString::from_str("kCFRunLoopDefaultMode")` which is the string name, not the constant. May need `NSDefaultRunLoopMode` from Foundation.
- **Codex API goes through OpenRouter** — Sometimes gets 401 errors. Retry if it fails.
- **The notification icon (menubar_icon.png) was inverted to white** for the dark menu bar. If switching to template image mode (which auto-adapts), use the original black icon.
- **`image.setTemplate(true)`** is set in `load_icon()` — this means the black SVG icon should auto-invert for light/dark. The white-inverted PNG may display wrong in light mode. Consider regenerating from the original black SVG.
- **Duplicate `#[derive(Args)]`** bug keeps appearing when adding new Args structs. Always check for it.
- **The `sync_received_account` method exists in sync.rs** but daemon.rs has its own `daemon_sync` function that duplicates the logic. Consider consolidating after menu works.
